# 人工复筛与 CSV 导入

人工复筛是 LitNexus 的最终人工决策环节：软件负责下载、合并、翻译和 AI 初筛；**是否纳入由人工填写的 `include` 决定**。本页定义当前 Mac 版本已经执行的 CSV 数据契约与异常处理。

> **实现状态（2026-07-12）**：数据页已提供小问号说明和“预检 → 明确确认 → 写入”两步导入。导入器只认 `epmc_id`、`include`、`tags`，默认不覆盖已有人工标注。

## 操作流程

1. 在“数据”页选择要导出的范围，导出 CSV。
2. 用表格软件复筛；保留 `epmc_id`，只编辑 `include` 和 `tags`。
3. 保存为 CSV（UTF-8 最稳妥），在“导入复筛结果”中选择文件。
4. 先阅读预检：错误必须修复；未匹配行和已有标注冲突会清楚列出。
5. 确认后才写入数据库。写入前软件会创建数据库备份。

## 只认三列

导入器只使用以下三列。其他列可以保留为阅读上下文，但即使被修改也**绝不会覆盖**数据库中的标题、摘要、期刊、翻译或 AI 答案。

| CSV 列 | 是否必须 | 含义与规则 |
|---|---:|---|
| `epmc_id` | 是 | 唯一匹配键。必须保留列名和每篇文章的值；不能用标题、DOI 或行号替代。 |
| `include` | 否 | 人工纳入结果。只接受英文 `yes` 或 `no`；首尾空格和大小写会被规范化，例如 ` YES ` 写为 `yes`。 |
| `tags` | 否 | 人工自由标签，可写任意简短文本，例如 `核心`、`方法学`、`待讨论`。 |

`include` 与 `tags` 至少应有一列存在；没有任何可写列的 CSV 不能导入。

### `include` 的唯一合法值

| 填写 | 含义 | 导入后状态 |
|---|---|---|
| `yes` | 人工决定纳入 | 纳入 |
| `no` | 人工决定排除 | 排除 |
| 留空 | 本次不改变原有人工标注 | 保持原值 |

不要填写 `是` / `否`、`Y` / `N`、`1` / `0`、`maybe`、`N/A` 或自定义文字。它们会被预检报错，而不是由软件猜测含义。

空白的语义尤其重要：它表示“**不改动**”，不是“改回待复筛”。将已有结论重置为待复筛，必须使用软件提供的明确重置操作；不能借由空单元格实现。

### 示例

下面的 `title` 只是供人阅读；导入时不会读取它。

```csv
epmc_id,include,tags,title
12345678,yes,核心,"A study to include"
PMC1234567,no,不符合范围,"A study to exclude"
MED9876543,,待讨论,"Do not change include this time"
```

## 预检如何处理异常行

选择文件后，软件会先进行预检，不直接写数据库。预检会把每一行归入下列类别，并在确认页显示数量和明细。

| 情况 | 预检结果 | 会不会写入 |
|---|---|---:|
| `epmc_id` 存在、值合法、至少一项人工字段有值 | 可导入 | 是，受覆盖规则约束 |
| 完全空白行 | 忽略 | 否 |
| 多出的列，或只改了标题/摘要/AI 列 | 忽略这些列 | 仅按三列规则写入 |
| 数据库中找不到该 `epmc_id` | 未匹配警告 | 否 |
| 有 `include` 或 `tags`，但 `epmc_id` 为空 | 错误 | 否，修复后才能确认 |
| 同一 `epmc_id` 在两行都填写了 `include` 或 `tags` | 错误 | 否，必须去重以避免“最后一行覆盖前一行”；仅供阅读的空白重复行会忽略 |
| `include` 不是 `yes` / `no` | 错误 | 否，必须改正 |
| 没有 `epmc_id` 列 | 错误 | 否 |
| 没有 `include`、也没有 `tags` 列 | 错误 | 否 |

“错误”会阻止本次写入；“未匹配”是可见的警告，合法且匹配的其他行可以在确认后继续导入。任何异常都不能被静默吞掉。

## 覆盖与冲突

人工复筛导入默认采取**只填补空值、不覆盖已有人工标注**的策略：

- 数据库中的 `include` 为空，CSV 给出 `yes` / `no`：可以写入。
- 数据库中的 `tags` 为空，CSV 给出标签：可以写入。
- 数据库已有 `include` 或 `tags`，CSV 试图写入不同值：预检列为“覆盖已有标注”。默认不写入。
- 用户如确实要覆盖，必须在确认页显式开启“允许覆盖已有人工标注”，并再次确认冲突数量。

这条规则只约束人工复筛 CSV；它不等同于“导入另一个数据库”的合并策略。整库导入的“跳过已有 / 仅填补空缺”等规则见[数据页说明](page-help.md#data-page)。

## 表格软件的建议

- 不要排序后只复制一列再粘贴到另一份 CSV；这很容易让结果与 `epmc_id` 错位。
- 过滤、排序、隐藏列都可以，但保存时必须保留 `epmc_id` 表头和对应的值。
- 若 Excel 自动改写很长的 ID、科学计数法或编码，先关闭自动格式化，或使用 LibreOffice / Numbers 后再检查导出文件。
- 标签中允许逗号、引号和换行；LitNexus 导出的 CSV 使用标准转义，重新保存时应选 CSV 格式。
- 建议一次导入一批明确的复筛结果，并保留原始导出文件；数据库自动备份不是替代你自己的版本管理。

## 数据页小问号的简版文案

数据页“导入复筛结果”旁的 `?` 应概括以下内容：

> 只按 `epmc_id` 匹配，只读取 `include` 与 `tags`。`include` 只能填 `yes` 或 `no`；留空表示本次不改动。其他列可供阅读，修改后不会写回。导入前会预检重复 ID、非法值和未匹配行；已有人工标注默认不覆盖。

完整规则与异常处理见本页。
