# 页面说明与帮助

Mac 桌面端在每个主页面标题旁提供 `?` 帮助。悬停可快速了解用途；点击后可固定查看该页的操作说明。本页是帮助浮层的完整版本，适合在需要时从应用内继续阅读。

## 运行

**用途**：按顺序执行下载、合并、翻译与分类，或从更多操作中单独运行某一步。

1. 设定抓取范围和时间窗口。
2. 选择“开始运行”。涉及 AI 的翻译和分类会在首次运行前确认，避免意外消耗额度。
3. 在“执行路径”查看每一步的状态、数量、耗时与警告。
4. 在“运行记录”查看本次运行的摘要；需要排障时再展开技术详情或复制诊断。

下载只产生原始文件，合并才会写入数据库；翻译和分类会调用当前项目选定的 AI 方案。中止会在安全检查点生效，已完成的工作会保留。

## 数据

**用途**：查看库内数量，导出 CSV 供人工复筛，导回应答结果，以及备份或导入数据库。

- CSV 导出后可在表格软件中编辑人工标注列，再导回本项目。导入复筛结果旁的 `?` 会提示“只以 `epmc_id` 匹配，只读取 `include` 与 `tags`；`include` 只能填 `yes` 或 `no`，留空表示本次不改动”。
- 标题、摘要、期刊、翻译和 AI 列是只读上下文，即使在 CSV 中改动也不会写回数据库。
- 导入会先预检重复 ID、非法值、未匹配行与覆盖冲突；已有人工标注默认不覆盖。完整契约见[人工复筛与 CSV 导入](manual-review.md)。
- 数据库导入和清空前都会创建备份；清空只删除文章数据，保留项目配置和列结构。

## 统计

**用途**：先阅读项目的基础事实，再按需检查提示词和期刊清单所呈现的线索。统计只读取本项目数据库；刷新不会重新下载文献或调用 AI。

“总览”始终显示，作为当前项目状态的固定参照。总览以下的卡片分为基础分布和高级评估：默认展开“年代分布”和“期刊分布”，其余卡片默认收起。每张卡片都可单独展开、收起和调整顺序；这些阅读偏好保存在当前设备，不会写入项目数据。

点击“编辑”后，各卡片会暂时收起内容，但仍保持各自独立的卡片边界；可直接拖动卡片调整顺序。保存时会恢复进入编辑前的展开状态。

### 基础分布

#### 年代分布

年代图用于观察收录时间范围和不同年份的筛选结果。

- 切换到**人工复筛结果**时，图例只反映图中的待复筛、纳入和排除状态。
- 切换到某个问题时，选项会以“问题 · 问题名称”标识，图例也同步只显示该问题的“是 / 否 / 未分类”等结果，不与复筛状态混用。
- 当纳入文章比例很低时，可使用适合比较小数量级的刻度，使小段数据不被较大的总量淹没；线性刻度仍可用于查看绝对量级。

同一张卡片中的“来源构成”使用原始来源代码（例如 `MED`、`PPR`）展示。来源标题旁的 `?` 用于解释代码含义；它们描述文献来源类型，不代表纳入或排除结论。

#### 期刊分布

期刊分布展示完整期刊集合，而非软件预先截取的 Top N。可以搜索全部期刊；表头各列均可点击排序，并可切换升序或降序。表中的收录、待复筛、纳入、排除及入选率都只是现有数据的汇总：没有人工复筛结果时，入选率应显示为未知，而不是被解释为低质量或不相关。

### 高级评估

高级卡片默认收起，避免把日常要看的语料事实与需要解释的分析线索混在一起。它们分为两类。

#### 提示词评估

- **分类结果分布**：按“问题 · 问题名称”查看该问题产生的分类结果。结果偏高或偏低都只是进一步检查提示词范围的信号，不等同于提示词正确或错误。
- **AI 与人工对照**：将 AI 的结果与人工复筛并列，使用中性的“AI 结果 / 人工复筛结果 / 不一致”表述，而不将某一类结果直接称为“AI 错误”。除逐个问题的对照外，也应支持查看跨问题的综合回看，例如所有问题均倾向纳入、但人工最终排除的文献集合。
- **对照结果导出**：可导出当前选定的对照集合，供外部分析或让 AI 协助起草新的筛选问题；导出不会更改数据库中的人工标注或 AI 结果。
- **检索词产出**：展示每条检索式带来的文献与纳入结果。这里的“重建检索渠道”是指扫描项目已有的合并文件，恢复“文章与命中检索式”的对应关系；不会重新下载文献，也不会改动文章、复筛或分类数据。

#### 期刊清单观察

这一部分展示期刊在当前数据中的收录、复筛和纳入表现，并区分“已在当前期刊清单中”和“未在清单中”的条目。它是观察清单覆盖与已有结果的工具，不应把任何条目直接表述为“建议加入”或“建议删除”，也不会自动修改期刊配置。可用的完整结果应可继续查看，而不是只保留若干条示例。

期刊入选率、提示词对照和期刊清单观察都依赖已有的人工复筛数据；样本很少、人工复筛尚未完成或问题定义变更时，应将它们视为待核验的线索，而非自动决策。

## 配置

**用途**：管理项目外观、检索列表、AI 方案、翻译与分类参数、项目位置与高级数据结构。

- 外观模式只保存在当前设备；项目强调色写入 `litnexus.toml`，可随项目迁移。
- AI 方案、分类问题、期刊清单与关键词检索式都会自动保存到项目配置；新项目以 `litnexus.toml` 为唯一主配置源。
- CSV 导出的范围和列只在“数据”页调整，并会自动记住为下次默认值；配置页不再重复放置导出设置。
- `include`、`tags` 是固定的复筛列。额外人工列属于“项目 → 数据结构（高级）”，不改变复筛 CSV 只写入 `include` / `tags` 的契约。
- 新增问题时，应明确选择只用于未来新文章，还是为历史文章补答；后者会消耗额外 AI 额度。
- 修改分类问题文本前，请阅读确认选项。语义变化时应新建版本并归档旧问题，不能把已有答案直接解释为新问题的答案；归档问题默认不进入以后的分类与当前统计。
- API Key 位于项目配置中。共享或上传项目目录前，应先移除敏感信息。
