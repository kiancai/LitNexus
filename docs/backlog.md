## 已完成

### 人工复筛安全导入与问题生命周期（2026-07-12）

- 数据页“导入复筛结果”已有 `?` 简明说明；完整规则见[人工复筛与 CSV 导入](guide/manual-review.md)。
- CSV 只以 `epmc_id` 匹配，只读取 `include` 与 `tags`；`include` 只接受 `yes` / `no`，空白不改变原值。
- 导入先预检：重复 ID、非法值、缺 ID、缺必要列会阻止写入；未匹配 ID 和覆盖冲突可见。
- 已有 `include` / `tags` 默认不覆盖；用户可显式开启覆盖并再次确认。
- 问题默认归档而非硬删除；新问题默认只作用于创建后新入库文章，补答历史文章必须显式开启。

### 检索后按期刊 / 类型剔除结果

**状态**：未实现（2026-07-07 记入）

**场景**：关键词检索会捞到一些影响因子很低、质量不佳但文章量很大的期刊，希望能在检索完成后把它们从结果里剔掉，避免低质大量内容淹没优质结果。

**功能要求**：
- 按「期刊 list」剔除：给定一组期刊名（或期刊 ISSN），删除结果中属于这些期刊的文章。
- 按「类型」剔除：例如剔除预印本（`source = PPR`）等。
- 这些都是**可选能力，默认不开启**；用户需要时再打开，不用亦可。

**实现备注**：
- 与「检索渠道追踪 / `article_terms` 关联表」待办可协同（前者记录每篇文章来自哪条检索词/哪种渠道，便于按渠道或按期刊维度做后续筛选）。
- 数据层已有 Europe PMC `source` 代码与期刊字段，剔除逻辑可作为 merge/检索后的一道过滤步骤；注意只作用于「关键词检索」结果，期刊检索结果不应被自家期刊 list 误伤。
