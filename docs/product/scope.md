# 目标与边界

写清「做什么 / 不做什么」，避免功能和文档再次发散。

## 目标（In scope）

| 目标 | 说明 |
|------|------|
| 文献拉取 | 期刊列表 + 关键词检索式，经 Europe PMC API |
| 本地库 | SQLite 去重、可迁移的工作区 |
| AI 辅助 | 标题翻译；可配置多问题分类（结果写入库） |
| 人工闭环 | CSV 导出 → 复筛 → 导回标注 |
| 桌面优先 | 双击即用的原生客户端（Mac 必须；Windows 随后） |
| 轻量 | 目标体积小、不捆绑沉重运行时（相对 NiceGUI/WebView2 路径） |

## 非目标（Out of scope，至少现阶段）

| 非目标 | 说明 |
|--------|------|
| 替代全文阅读器 / 参考文献管理器 | 不做 Zotero/EndNote 级管理 |
| 共享跨语言引擎 | 不做「一份 Rust 核心两端绑」；采用各端重写 + 测试对齐（方案 A） |
| Linux 客户端 | 暂不做（可砍） |
| 服务端 / 账号体系 | 纯本地工作区；AI 走用户自备 API |
| 全文 PDF 自动下载与解析 | 当前以元数据 + 标题摘要为主 |
| 自动「决定是否重要」的黑盒产品 | AI 只做初筛；最终标记归人 |

## 设计原则（已定）

- **行为以测试为标尺**：Python 参考实现 + selftest；原生端 1:1 对齐行为，不追求共享代码。
- **工作区自包含**：配置、列表、库、下载、导出都在 vault 内。
- **桌面端不以环境变量为主**：Mac UI 用界面内 AI 多方案，即时持久化。
- **合并只处理新文件**：已合并 JSONL 移入 `downloads/_merged/`，避免重复劳动。

原则会随文档完善增补；有争议时先改本文档，再改代码。
