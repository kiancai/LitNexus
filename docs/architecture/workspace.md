# 工作区与配置

## 工作区目录结构

```text
<root>/
├── litnexus.toml    # 项目配置：检索列表、AI 方案、问题、下载参数、主题色
├── litnexus.db      # SQLite 数据库（WAL）
├── downloads/       # 原始 JSONL（含 _merged/）
└── exports/         # 导出的 CSV、对照结果等
```

这是用户的「vault」：备份这个目录，约等于备份项目状态。`litnexus.db-wal` 与 `litnexus.db-shm` 是 SQLite 运行时文件；若数据库正在被使用，不应只复制其中一个文件。优先使用应用内数据库备份／导出，或先完全退出应用再复制整个目录。

## 工作区如何被找到

Mac 桌面端通过项目选择页或首次向导创建、打开工作区。当前活动项目和最近打开列表只保存在本机的应用支持目录中；项目本身不会记录绝对路径，因此复制或移动整个工作区后仍可在另一台机器上打开。

桌面端不读取环境变量来决定工作区或 AI 配置。交互以应用内选择和工作区根目录的 `litnexus.toml` 为准。

## `litnexus.toml` 是主配置源

Mac 当前把期刊清单和关键词检索式与其他项目配置一起保存在工作区根目录的 `litnexus.toml`。概念上如下：

```toml
[download]
journals = ["Nature", "Bioinformatics"]
keywords = [
  "(microbiome OR microbiota) AND \"machine learning\"",
  "\"single cell\" AND (deep learning OR neural network)",
]
```

- `journals`：期刊下载通道使用的期刊名。
- `keywords`：关键词下载通道使用的 Europe PMC 检索式。
- 每一项是一条字符串；空行或以 `#` 开头的项可作为编辑时的分隔／注释，下载时会跳过。
- Mac 配置页会自动保存这两组列表。手工编辑 TOML 时，下一次重新打开项目会读取新的值。

### 旧列表文件的兼容性

早期项目可能含有 `journals.txt`、`keywords.txt` 或 `keywords/*.txt`。它们不是新项目的主配置：当 TOML 中缺少对应的 `download.journals` / `download.keywords` 时，客户端可把旧文件作为兼容性读取来源。之后一旦从应用保存配置，应以 `litnexus.toml` 中的数组为准。

不要同时把两个位置当作可编辑的真相来源，否则很难判断下一次运行到底采用哪套列表。

## 其他配置

- 分类问题：`[[classify.questions]]`；每项有稳定 `id`、显示昵称和问题文本。
- AI 方案：保存在 TOML 中，可在应用内添加、选择、删除和编辑，改动自动保存。
- 导出选项和自定义人工标注列：保存在 TOML 中。
- 项目强调色：可写入 `[theme].accent_hue`；浅色 / 深色 / 跟随系统是本机显示偏好，不写入项目。

API Key 属于项目敏感信息。共享、上传或提交工作区前，应移除 API Key 或改用不含密钥的副本。

## 配置、历史与生命周期

配置表示“**从现在开始要怎样运行**”；数据库表示“**过去发生过什么**”。因此，编辑期刊、检索式或 AI 问题不应暗中删除历史文献、人工复筛或历史 AI 答案。

### 分类问题

下列是已确定的数据语义；其中版本化存储会在后续数据库迁移中落地，详见[数据库](database.md#question-query-lifecycle)。

| 操作 | 此后流水线 | 历史数据 | 统计默认视图 |
|---|---|---|---|
| 新增问题 | 创建时明确选择“仅未来新文章”（默认）或“为历史文章补答”；仅未来模式记录当前入库边界 | 既有答案不改 | 当前问题集合中显示新问题；范围外历史文章不计作未分类 |
| 修改问题文本 | 语义变化时默认创建新问题／归档旧问题；不能悄悄把旧答案解释为新问题的答案 | 保留旧问题与答案 | 显示当前问题；归档问题保留在配置页 |
| 停用／删除问题 | 默认归档，停止以后 AI 分类 | 保留历史答案与理由 | 默认隐藏归档问题，不混入当前提示词统计 |
| 永久清除问题 | 仅高级维护操作；需自动备份和二次确认 | 才可能物理删除 | 不再显示 |

当前客户端仍使用问题动态列作为过渡实现；因此“永久删除”是风险操作。界面和数据库会逐步迁移到稳定 ID 与版本记录，迁移期间不应复用已删除问题的 ID。

### 期刊与关键词检索式

| 操作 | 此后下载 | 历史数据 |
|---|---|---|
| 新增 | 只影响后续下载 | 不回写、不删除已有文章 |
| 修改 | 当前只影响以后的下载；长期会变为稳定版本，而非覆盖历史检索式 | 原命中及其原检索式应保留 |
| 停用／删除 | 当前从列表移除，不再用于后续下载 | 不删除已下载文章或旧命中 |

当前检索渠道关联以文章和检索式文本为主；长期会迁移到带稳定 ID、版本和下载批次的记录。详细目标见[数据库](database.md#question-query-lifecycle)。
