# 架构总览

## 顶层设计

三层，不要混：

1. **产品语义**（与语言无关）— 工作区、流水线、库表意图  
2. **实现** — `mac/` 主线，`win/` 随后  
3. **文档** — `docs/` 只服务文档站网页  

```text
        文档 docs/
            │ 约束
      ┌─────┴─────┐
      ▼           ▼
     mac/        win/
    当前产品     下一阶段
```

## 仓库目录（入库）

```text
LitNexus/
├── docs/          # 仅网页文档源
├── mac/           # Mac 原生
├── win/           # Windows 占位
├── mkdocs.yml
└── README.md
```

本地可能还有不入库的目录（如旧 `python/`、本机 `AGENTS.md`），不参与产品结构。

## 产品分层（概念）

```text
UI（mac / 未来 win）
  → 流水线
    → EPMC · 数据库 · AI · CSV
      → 工作区磁盘契约
```

各端各自实现；磁盘契约与流水线语义一致。详见 [多端策略](platforms.md)。
